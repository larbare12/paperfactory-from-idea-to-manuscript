## 功能
实验方案检查、指标选择、对比方法建议

## 输入
- 研究问题
- 可用数据/资源
- baseline 方法列表
- 目标期刊/会议的实验要求

## 输出
- 实验方案检查清单
- 评估指标建议
- 对比方法选择
- 消融实验设计
- 显著性检验建议
- 实验完整性评估

## 使用场景
1. 设计实验方案
2. 检查实验完整性
3. 补充缺失实验
4. 回应审稿人关于实验的质疑

## 实验设计检查清单

### 数据集
- [ ] 数据集是否公开可获取？
- [ ] 数据集规模是否足够？
- [ ] 训练/验证/测试划分是否合理？
- [ ] 是否存在数据泄露风险？
- [ ] 数据预处理是否标准化？

### 评估指标
- [ ] 主要指标是否与领域标准一致？
- [ ] 是否包含多个互补指标？
- [ ] 指标计算方式是否明确？
- [ ] 是否报告标准差/置信区间？

### 对比方法
- [ ] 是否包含领域 SOTA 方法？
- [ ] 是否包含经典 baseline？
- [ ] 对比方法是否使用相同设置？
- [ ] 是否公平比较（相同数据、相同评估）？

### 消融实验
- [ ] 是否验证关键组件贡献？
- [ ] 消融设计是否系统全面？
- [ ] 是否分析超参数敏感性？

### 显著性检验
- [ ] 是否进行统计显著性检验？
- [ ] 检验方法是否恰当（t-test/bootstrap）？
- [ ] 是否报告 p-value？

### 误差分析
- [ ] 是否分析失败案例？
- [ ] 是否按类别/难度分析性能？
- [ ] 是否可视化典型结果？

### 计算资源
- [ ] 是否报告硬件配置？
- [ ] 是否报告训练/推理时间？
- [ ] 是否报告内存占用？

## 实验设计流程

### Step 1: 目标明确
明确实验要验证的假设

### Step 2: 数据集选择
选择合适的数据集，考虑规模、难度、领域代表性

### Step 3: 指标设计
设计主指标和辅助指标

### Step 4: 对比方法确定
选择有代表性的 baseline

### Step 5: 实验配置
确定超参数、随机种子、重复次数

### Step 6: 消融设计
设计验证各组件贡献的实验

### Step 7: 显著性检验
设计统计检验方案

### Step 8: 误差分析
设计案例分析和可视化方案

## 评估指标选择指南

### 分类任务
- 主要：Accuracy, F1-score
- 辅助：Precision, Recall, AUC-ROC

### 回归任务
- 主要：MSE, MAE
- 辅助：R², Pearson correlation

### 生成任务
- 主要：BLEU, ROUGE, METEOR
- 辅助：人工评估、多样性指标

### 检索任务
- 主要：Recall@K, MRR, NDCG
- 辅助：Precision@K

## 对比方法选择原则

### 必须包含
1. **经典方法**：领域奠基性工作
2. **SOTA 方法**：当前最佳性能
3. **近邻方法**：与本文方法最相关
4. **简单 baseline**：证明方法有效性

### 公平比较
- 相同数据集划分
- 相同评估指标
- 相同训练设置（如可能）
- 报告多次运行的平均和标准差

## 消融实验设计

### 组件消融
```
Full model: A + B + C
  -w/o A: B + C
  -w/o B: A + C
  -w/o C: A + B
  -w/o A,B: C
  -w/o A,C: B
  -w/o B,C: A
```

### 超参数敏感性
- 关键超参数的范围测试
- 学习率、batch size、正则化系数等
- 可视化性能随参数变化曲线

## 显著性检验

### 常用方法
- **t-test**：两组均值比较
- **bootstrap**：置信区间估计
- **Wilcoxon signed-rank**：非参数检验

### 报告规范
- 报告检验方法
- 报告 p-value
- 显著性水平（通常 p < 0.05）

## 常见实验缺陷

| 缺陷 | 影响 | 修正 |
|-----|------|------|
| 数据泄露 | 性能虚高 | 严格划分训练/测试 |
| 对比不公平 | 结论不可信 | 相同设置、多次运行 |
| 指标单一 | 评估不全面 | 多指标综合评估 |
| 缺乏消融 | 贡献不清晰 | 系统消融实验 |
| 无显著性检验 | 结果不可靠 | 统计检验 |
| 无误差分析 | 理解不深入 | 案例分析 |
