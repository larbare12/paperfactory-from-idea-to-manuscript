# M3 论证设计模块

## 功能
设计核心论点、证据链、反驳预判

## 输入
- 核心 claim（一句话）
- 已有证据（数据/实验/理论）
- 目标读者
- 潜在反对意见（可选）

## 输出
- 论证结构图（论点-证据-推论）
- 潜在漏洞识别
- 反驳预判及回应策略
- 理论框架建议
- 证据链完整性检查

## 使用场景
1. 设计论文核心论证
2. 准备答辩/演讲的论证逻辑
3. 回应审稿人质疑
4. 设计实验验证策略

## 论证设计流程（基于推理架构提取思想）

### Step 1: 核心论点提取
明确论文的 main claim，用一句话概括

### Step 2: 证据链构建
识别支撑 claim 的关键证据，建立逻辑链条

### Step 3: 推理模式识别
确定论证类型：演绎/归纳/类比/因果

### Step 4: 评估阈值设定
明确什么程度的证据可以支持 claim

### Step 5: 引用逻辑设计
选择关键文献支撑论证

### Step 6: 反驳预判
识别可能的反对意见，准备回应

### Step 7: 边界界定
明确 claim 的适用范围和限制

### Step 8: 论证整合
生成完整的论证框架

## 论证结构模板

### 标准结构
```
Claim: 本文提出的方法 X 在任务 Y 上优于现有方法 Z

Evidence 1: 在数据集 A 上，X 的准确率比 Z 高 15%
  → Support: 实验结果（Table 1）
  
Evidence 2: 消融实验显示组件 B 是关键贡献
  → Support: 消融实验（Table 2）
  
Evidence 3: 复杂度分析显示 X 与 Z 相当
  → Support: 理论分析（Section 3.4）

Warrant: 更高的准确率 + 可接受的复杂度 → 更好的方法

Backing: 领域共识认为准确率是首要指标

Qualifier: 在数据集 A 的条件下

Rebuttal: 如果考虑实时性要求，X 可能不如 Z
  → Response: X 的推理时间也在可接受范围内（Table 3）
```

## 常见论证模式

### 模式 1: 新方法优于 baseline
```
Claim: 方法 X 优于现有方法
Evidence: 在多个数据集上的实验对比
Support: 消融实验验证关键组件
Counter: 复杂度/可解释性/泛化性讨论
```

### 模式 2: 理论分析指导实践
```
Claim: 理论 T 可以解释现象 P
Evidence: 理论推导 + 实证验证
Support: 文献中的类似现象
Counter: 边界条件和例外情况
```

### 模式 3: 新任务/新数据集
```
Claim: 任务 T 具有重要意义
Evidence: 应用场景 + 现有方法的不足
Support: 初步实验结果
Counter: 与相关任务的区分度
```

## 论证检查清单

- [ ] Claim 是否清晰明确？
- [ ] 证据是否充分支撑 claim？
- [ ] 推理过程是否有逻辑跳跃？
- [ ] 是否考虑了替代解释？
- [ ] 是否界定了适用范围？
- [ ] 是否回应了潜在反驳？
- [ ] 引用是否恰当支撑论证？
