初衷

天下研究生，苦小论文久矣。paperfactory 想做一件事——把从 idea 到 manuscript 的整条生产线自动化。你负责想 idea，剩下的交给流水线。

核心特性

六大硬约束协议,组合起来形成完整的论文质量护栏。

反幻觉硬约束

Tier 0 校验(DOI 反查 + Levenshtein ≥ 0.70),识别 5 类幻觉:TF / PAC / IH / PH / SH。三层验证 + 红线清单,任一触发立即 STOP。

Venue Quality 二道筛

反幻觉只保证文献真实,不保证够顶。venue_lookup.sh 自动判级,预印本(NBER WP / arXiv / SSRN)不能当方法论 baseline。

三段式文献工作流

广搜 → 筛选 → 收集自动入 manifest + OA PDF 下载 → 用户补全闭源 → prune 找不到的。

Anti-Leakage Protocol

M6 写作铁律:数据点 / 引用 / 统计必须有本地证据,缺则插 [MATERIAL GAP],final draft 拒绝输出含 GAP。

Devil's Advocate Pass

M5 autonomous 强制:每个 core claim(3-7 个)跑 Attack / Rebuttal / Concession 三轮,反 cascade concession。

插件式 Hooks

SessionStart / PreToolUse / PostToolUse / Stop 自动激活;非 paper 项目静默 exit 0,不污染其他 Claude session。

快速开始

三步上手,从安装到第一次跑 M0 dashboard。

  1. 安装 plugin

    把整个 plugin 目录放到 Claude Code 的 plugin 路径(参见 Claude Code 文档),或在本目录直接启动 Claude Code session。

  2. 在论文项目里跑 init

    cd /path/to/your-paper-project
claude  # 启动 session
> /paperfactory:init

    会自动:检查 git 仓库 + 工作区 → 检查 S2 API key 凭据 → 初始化 relate-work/ + manifest.jsonl → 启用 hooks → 跑 M0 仪表盘首次扫描 → 给出下一步建议。

  3. 按 M0-M9 工作流推进

    通过 @skill-name 路由到对应阶段(详见下方路由表)。

M0-M9 路由

每个阶段是独立 skill,可被 @ 单独激活;citation-search 是横切子 skill,被其他 9 个 skill 内部委托调用。

架构

独立 skill + 横切子 skill + plugin-enforced hooks 的三层组合。

完整目录树 
.claude-plugin/

│   └── reference/           # 协议:literature-research / anti-hallucination / venue-quality / manifest-schema
├── m0-dashboard/            # 项目仪表盘(横切常驻)
├── m1-topic/                # 选题诊断
├── m2-literature/           # 文献管理
├── m3-experiment/           # 实验设计
├── m4-structure/            # 结构规划
├── m5-argument/             # 论证设计 + DA 协议
├── m6-writing/              # 写作辅助 + Anti-Leakage
├── m7-final-check/          # 投稿前总检
├── m8-peer-review/          # 同行评审仿真
└── m9-compliance-check/     # 合规与伦理检查

commands/

凭据要求

# 在你的论文项目根创建 .env
cp ${CLAUDE_PLUGIN_ROOT}/.env.example .env

# 在 https://www.semanticscholar.org/product/api 申请 S2 API key
# 填入 .env:
S2_API_KEY=Bearer xxxxx

# multi mode 额外需 Python:
pip install -r ${CLAUDE_PLUGIN_ROOT}/requirements.txt

/paperfactory:init 会自动跑 verify_config.sh 检查。
