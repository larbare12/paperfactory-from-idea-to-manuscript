  研二还没发小论文？我写了个开源工具，把整条论文生产线自动化了。


  故事从研二上学期开始说起。导师每周催一次进度，我打开 Overleaf，
脑子一片空白，根本不知道从哪开始。

  一直到 OpenClaw 的问世。那时我只是用它来辅助完成我的小论文。直到
四月，一个想法在我的脑子里炸开 —— 既然现在的 Agent 这么强，可以自己
写代码、自己写论文，为什么不直接让它自己干？

  后来我花了两个月，把这些问题一个个拆开，写了个 Claude Code 插件叫
paperfactory。设计的初衷很简单：

      你付出 idea，Agent 帮你实现。


  ▎ 具体能干嘛
  ▎
  ▎ 选题诊断（M1）—— 放进一个 idea，它帮你做可行性评估。研究 gap
  ▎   存不存在？方法适不适合？跟已有文献撞不撞车？
  ▎
  ▎ 文献检索 + 反幻觉（M2）—— 这个是最狠的。它不只帮你搜论文，还
  ▎   会逐条做 DOI 反查。模型编造出来的假引用、把 arXiv 预印本当顶
  ▎   刊引用，统统给你揪出来。这功能是我自己吃过大亏之后硬加上的
  ▎   —— 一篇 NBER working paper 被错当成顶刊方法论 baseline，
  ▎   整层逻辑推倒重写。
  ▎
  ▎ 实验设计（M3）—— 根据你的研究类型，推荐实验范式、样本量估算、
  ▎   对照组设计。
  ▎
  ▎ IMRAD 结构化（M4）—— 自动出大纲，Introduction / Methods /
  ▎   Results / Discussion 骨架一套生成。
  ▎
  ▎ 论证红队（M5）—— AI 自己扮演审稿人，对你的每个核心论点做三轮
  ▎   攻击：Attack / Rebuttal / Concession。反驳力不够 4 分的论点，
  ▎   建议软化或降级。
  ▎
  ▎ 写作辅助 + 证据铁律（M6）—— 所有数据点必须有本地证据支撑。
  ▎   缺证据的地方自动插 [MATERIAL GAP] 标记，没填完不准出终稿。
  ▎
  ▎ 投稿前总检（M7）—— 引用完整性、格式规范、数据可复现性，
  ▎   一条龙审计。
  ▎
  ▎ 审稿仿真（M8）—— 模拟 3 个审稿人视角，提前给你一份 review
  ▎   report。
  ▎
  ▎ 合规检查（M9）—— AI 披露声明、伦理审查、PRISMA-trAIce 协议等。


  说白了，9 个环节，从「我有个 idea」到「可以投了」，一步一步带着你走。

  免费。开源。一行命令快速安装。

  作者有一个朴素的愿望：希望天底下的研究生，能够免去科研小论文的苦恼。
