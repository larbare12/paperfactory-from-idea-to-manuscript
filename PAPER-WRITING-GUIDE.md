# 论文写作指南

> 基于 OpenClaw 项目 Prompt 设计经验整理的学术写作规范

---

## 第一部分：从 Prompt 设计中学到的写作原则

### 1.1 明确的目标导向

**项目经验**: 每个 Skill 的 description 必须清晰说明"何时使用"。

**论文应用**:
- 引言部分明确研究问题和目标
- 每个章节开头说明该部分的目的
- 避免模糊表述

**对比示例**:
```
❌ 模糊: "本文研究机器学习在医疗领域的应用"
✅ 明确: "本文旨在解决医疗影像诊断中深度学习模型可解释性不足的问题，
         通过提出 XX 方法，在 YY 数据集上验证 ZZ 假设"
```

### 1.2 结构化表达

**项目经验**: SKILL.md 使用标准结构（Frontmatter + Body + References）。

**论文标准结构（IMRAD）**:
```
├── 摘要（Abstract）
│   └── 浓缩版全文：背景→方法→结果→结论
│
├── 引言（Introduction）
│   ├── 研究背景
│   ├── 问题陈述
│   ├── 研究目标
│   └── 贡献概述
│
├── 相关工作（Related Work）
│   ├── 领域概述
│   ├── 现有方法分类
│   └── 本文与已有工作的区别
│
├── 方法（Methodology）
│   ├── 问题定义
│   ├── 算法/模型描述
│   ├── 实现细节
│   └── 复杂度分析
│
├── 实验（Experiments）
│   ├── 数据集描述
│   ├── 评估指标
│   ├── 对比方法
│   └── 实验设置
│
├── 结果（Results）
│   ├── 主要结果
│   ├── 消融实验
│   └── 误差分析
│
├── 讨论（Discussion）
│   ├── 结果解释
│   ├── 局限性
│   └── 启示
│
└── 结论（Conclusion）
    ├── 主要发现
    ├── 理论/实践意义
    └── 未来工作
```

### 1.3 渐进式信息呈现

**项目经验**: 先给概览，再给细节；复杂内容拆分到 references。

**论文应用**:
- 先呈现主要结论，再展开详细论证
- 复杂公式/算法放入附录
- 使用"详见第X节"引导读者

**示例结构**:
```markdown
## 3. 方法

### 3.1 概述
我们的方法包含三个主要组件：...（一句话概括）

### 3.2 详细设计
#### 3.2.1 组件A
...（详细描述）

#### 3.2.2 组件B
...（详细描述）

### 3.3 复杂度分析
详见附录A。
```

## 第二部分：用词与语言风格

### 2.1 学术写作禁用词

| 避免使用 | 替代方案 | 原因 |
|---------|---------|------|
| "我觉得" | "实验结果表明" | 主观→客观 |
| "很明显" | "数据显示" | 断言→证据 |
| "非常好" | "显著优于" | 模糊→量化 |
| "可能大概" | "在XX条件下" | 含糊→具体 |
| "这个东西" | "该算法/模型" | 口语→专业 |
| "我们做了" | "我们实施/执行" | 口语→正式 |

### 2.2 推荐用词风格

**✅ 推荐使用的表达**:
```
- "基于..."（Based on...）
- "结果表明..."（The results demonstrate...）
- "相比之下..."（In contrast...）
- "值得注意的是..."（Notably...）
- "具体而言..."（Specifically...）
- "由此推断..."（This suggests that...）
```

**❌ 避免使用的表达**:
```
- "我觉得..." / "我认为..."
- "很显然..." / " obviously..."
- "大家都知道..."
- "这个那个..."
- "挺多的" / "不少"
```

### 2.3 时态使用规范

| 章节 | 时态 | 示例 |
|-----|------|------|
| 引言/相关工作 | 现在时 | "XX is a critical problem..." |
| 方法描述 | 过去时 | "We implemented..." / "The model was trained..." |
| 实验结果 | 过去时 | "The model achieved..." / "Accuracy increased..." |
| 结论/讨论 | 现在时 | "These findings suggest..." |

## 第三部分：写作流程规范

基于项目中 Skill 开发的 6 步流程，类比论文写作流程：

```
Step 1: 理解问题（Understanding）
        - 明确研究问题
        - 收集相关文献
        - 确定贡献点
        - 输出：研究问题陈述

Step 2: 规划内容（Planning）
        - 设计论文结构
        - 确定实验方案
        - 准备图表规划
        - 输出：大纲 + 图表草图

Step 3: 撰写初稿（Drafting）
        - 从方法/实验开始（最容易写）
        - 最后写引言（需要全文视角）
        - 摘要最后写
        - 输出：完整初稿

Step 4: 迭代修改（Iterating）
        - 自我审阅
        - 同行评审
        - 语言润色
        - 输出：修改稿

Step 5: 格式检查（Packaging）
        - 检查引用格式
        - 图表规范
        - 字数/页数要求
        - 输出：定稿

Step 6: 投稿/发布（Publishing）
        - 选择合适期刊/会议
        - 准备 Cover Letter
        - 回应审稿意见
        - 输出：投稿
```

## 第四部分：图表与数据呈现

### 4.1 表格规范

**基本要求**:
- 三线表（顶线、栏目线、底线）
- 表头清晰，单位明确
- 数据对齐（数字右对齐，文本左对齐）

**示例**:
```markdown
| 方法 | 准确率(%) | F1分数 | 训练时间(h) |
|-----|:--------:|:-----:|:----------:|
| 方法A | 85.2 | 0.823 | 2.5 |
| 方法B | 87.5 | 0.856 | 3.2 |
| 方法C | 89.1 | 0.878 | 4.1 |

表1: 不同方法的性能对比（数据集：XX，最佳结果加粗）
```

### 4.2 图表设计原则

1. **自明性**: 图表标题应说明内容，无需看正文即可理解
2. **一致性**: 相同类型的图使用相同的颜色、符号
3. **清晰度**: 字体大小、线条粗细适合打印（建议≥8pt）
4. **准确性**: 坐标轴标注、单位、误差棒不可少

### 4.3 图片规范

- 分辨率：≥300 dpi（印刷）
- 格式：矢量图优先（PDF/EPS），位图用 PNG（非 JPG）
- 颜色：考虑黑白打印的可读性

## 第五部分：引用与参考文献

### 5.1 引用格式要点

**直接引用**（使用原文）:
> "According to Smith et al. [1], 'deep learning has revolutionized...' (p. 45)."

**间接引用**（转述）:
> Smith et al. [1] demonstrated that deep learning approaches...

**避免过度引用**:
- 一句话不宜超过 2-3 个引用
- 避免连续多句都以引用开头

### 5.2 参考文献管理

**推荐工具**:
- Zotero（免费，推荐）
- EndNote
- Mendeley

**参考文献必备字段**:
```
期刊文章：
- 作者（完整姓名或标准缩写）
- 年份
- 文章标题
- 期刊名称（全称或标准缩写）
- 卷号(期号): 页码
- DOI

会议论文：
- 作者
- 年份
- 文章标题
- 会议名称
- 页码
- 地点
```

## 第六部分：常见错误与避免方法

| 错误类型 | 错误示例 | 修正 |
|---------|---------|------|
| **模糊主语** | "It is important to..." | "Accurate prediction is critical for..." |
| **冗余表达** | "In order to..." | "To..." |
| **口语化** | "We did a lot of experiments" | "We conducted extensive experiments" |
| **被动滥用** | "The experiment was done by us" | "We conducted the experiment" |
| **缩写未定义** | "We used CNN" | "We used Convolutional Neural Networks (CNN)" |
| **模糊量化** | "The method is much better" | "The method improved accuracy by 15%" |
| **绝对化表述** | "This is the best method" | "This method outperforms existing approaches" |

## 第七部分：实用写作 Prompt 模板

### 7.1 论文大纲生成 Prompt

```
请帮我为以下研究主题生成论文大纲：

研究主题：[填入主题]
研究问题：[填入具体问题]
方法：[填入方法]
预期贡献：[填入贡献]

要求：
1. 按照 IMRAD 结构（Introduction, Methods, Results, And Discussion）
2. 每个章节列出 3-5 个要点
3. 标注每个部分的预计字数
4. 指出需要重点阐述的部分
5. 建议 2-3 个可能的实验设计
```

### 7.2 段落润色 Prompt

```
请润色以下段落，要求：
1. 保持学术语气
2. 增强逻辑连贯性
3. 消除冗余表达
4. 确保术语一致性
5. 检查时态正确性

原文：
[填入段落]

目标期刊/会议风格：[填入，如 ACL/NeurIPS/CVPR]
```

### 7.3 摘要撰写 Prompt

```
请基于以下内容撰写论文摘要（200-300字）：

研究背景：
研究问题：
方法概述：
主要结果：
研究意义：

要求：
1. 结构：背景→问题→方法→结果→意义
2. 使用第三人称
3. 避免引用和缩写
4. 关键词：3-5个
5. 突出核心贡献
```

### 7.4 相关工作撰写 Prompt

```
请帮我撰写相关工作部分，要求：

研究领域：[填入领域]
本文方法：[填入方法]
需要对比的方法：[填入方法列表]

结构：
1. 领域概述（1-2段）
2. 方法分类（每类1段）
   - 基于XX的方法
   - 基于YY的方法
3. 与本文工作的对比（1段）
   - 现有方法的局限性
   - 本文的改进点

引用要求：
- 引用关键 baseline 方法
- 引用近3年的相关工作
- 引用领域经典论文
```

### 7.5 实验设计检查 Prompt

```
请检查以下实验设计是否完整：

实验内容：
[填入实验描述]

检查清单：
□ 数据集是否公开/可复现
□ 评估指标是否标准
□ 对比方法是否充分
□ 消融实验是否完整
□ 显著性检验是否进行
□ 误差分析是否包含
□ 计算资源是否说明

请指出缺失的部分并给出建议。
```
